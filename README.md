# Проект Bastion

**Создание децентрализованной P2P социальной сети по типу Reddit, совмещённой с YouTube. Используется максимальное сжатие, чтобы обнаружение было очень сложным.**


## Сжатия

- Сжатие текста и HTML/CSS - brotli (макс. степень)
- Фото - AVIF (AV1) / JPEG XL; Максимум - 1080p (16:9) или эквивалент
- Видео - AV1; качество и частота сжаты настолько, чтобы восстановить до 1080p@60fps. ~~WebNN + WebGPU~~ (TPM не вызвать нормально из браузера, нужно приложение, см. ниже). Приоритетно используется NPU, далее GPU, в последнюю очередь - CPU.


## Модель клиентов

- **Приложение**. Умеет и хранить данные, и апскейлить.
- **HomeLab**, контейнер для работы с локалкой. Отвечает ***целиком*** за хранение и шифрование, может апскейлить (опция). При репорте контента имеет больший вес.
- **Сайт**, поднятый на HomeLab. Имеет доступ лишь из локально сети, ***не умеет*** хранить, апскейл *приоритетно* на HomeLab, иначе через **WebNN + WebGPU**.


## Система хранения


Около 5 реплик данных нужно хранить для стабильного доступа. На 1 млн. символов, 1000 фото, 100 видео (по 10 минут) потребуется 15 ГБ. На 5 реплик - 75 ГБ. HomeLab помогут сниизить до 50 ГБ и ниже. 


## Система шифрования

Контент распространяется в зашифрованном виде, блоками (посты). Хранение происходит в кеше в зашифрованном виде, расшифровка **ТОЛЬКО** в ОЗУ, никакой записи на диск. 

Есть таблица **файл - идентификатор ключа**, уникальная для кеша устройства.

Есть общая для всех таблица **идентификатор ключа - ключ**, но полная её версия может хранится только там, где нет ни единого блока. Блок и ключ - взаимоисключают друг друга на одной системе.

Но! По умолчанию пользователь не хранит пару ключ для блока, только если он сам захочет (личная копия). Это сделано, чтобы случайно не лежало ЦП и т.п. сразу с ключом, но при модерации не удалили полностью что-то относительно безобидное, но спорное. Полная база или требующийся для скаченых блоков кусок базы (TPM не бесконечный) данных **идентификатор ключа - ключ** хранится только в TPM.


## Модерация

Количество репортов будет похоже на график y = a\*log<sub>b</sub>x (a > 0, x >= 2); y = a\*x (0 <= x < 2), где x - число просмотров, a - коэффициент, b - основание логарифма.

Вес определяется аптаймом, кол-вом переданных файлов, пожертвованным пространством.

При получении достаточного числа репортов, происходит бан ключа. При бане ключа он искажается случайным шумом, идентификатор ключа остаётся, просто теперь имеет сломанный ключ. Личные копии исходного ключа никто не может затронуть.

Если валидного ключа нет при синхронизации ни в глобальной, ни в личной базе нет, блок удаляется.

Репорт отправляется всем доступным пользователям. Синхронизация происходит по нескольким ответам, берётся медиана.


## Аутендификация

Приватный/публичный ключ. Статистика хранится распределённо.
